## 왜 이걸 했나

CEO가 `/claude start-sdk` 시작 시 Opus 디폴트가 200K인 것을 발견. 1M 활성화 방법을 찾아서 시작 시 모델/컨텍스트를 선택할 수 있도록 기능화. 진행 중에 SDK의 `betas` 플래그, 모델 식별자 규칙, `modelUsage` 표시 버그 등이 드러나 연쇄적으로 수정.

## 변경 사항

### 1. `/claude start-sdk` 2단계 선택 UI

- Step 1: 모델 + 컨텍스트 윈도우 선택 (Opus 4.7 1M/200K, Opus 4.6 1M/200K, Sonnet 4.6 1M/200K, Haiku 4.5 200K)
- Step 2: 디렉토리 선택
- 파일: `src/discord/commands/claude.ts`, `src/discord/interactions/select-menus.ts`

### 2. 1M 컨텍스트 모델 식별자

- **잘못된 초기 시도:** `betas: ['context-1m-2025-08-07']` → Sonnet 4/4.5 전용 (곧 retire), Opus에는 영향 없음
- **정답:** 모델 ID 뒤에 `[1m]` 접미사 (`claude-opus-4-7[1m]`, `claude-opus-4-6[1m]`, `claude-sonnet-4-6[1m]`)
- Claude Code CLI `/model` 명령이 이 형식을 받는다는 스크린샷으로 CEO가 확인
- 테스트 스크립트로 실제 `contextWindow: 1000000` 반환 검증
- 파일: `src/discord/claude-sdk/claude-sdk-session-manager.ts`

### 3. 모델 표시 버그 수정 — Haiku로 잘못 표시되던 문제

- 증상: Opus로 시작했는데 `🤖 claude-haiku-4-5` 표시
- 원인: `Object.keys(message.modelUsage)[0]`로 첫 번째 키를 가져왔는데, SDK가 Haiku를 사이드카(compaction 등)용으로 먼저 등록하는 경우가 있음
- 수정: 토큰 사용량 기준으로 정렬, 최다 사용 모델을 primary로. 추가로 모든 모델을 2번째 줄에 표시
- 표시 형식:
  ```
  🟢 14% ctx (28.4k/1.0M) · $0.1790 · turn 1
  🤖 claude-opus-4-7[1m]: 28.4k · claude-haiku-4-5: 2.1k
  ```
- 파일: `src/discord/claude-sdk/claude-sdk-session-manager.ts`, `src/discord/claude-sdk/claude-sdk-handlers.ts`

### 4. 세션 재시작 시 모델 유지

- 증상: Opus 4.7 (1M)으로 시작한 세션이 봇 재시작 후 200K로 fallback
- 원인: lazy resume 시 `model` 파라미터를 전달하지 않아 SDK가 `claude-opus-4-7` (200K variant)로 해석
- 수정: `PersistedMapping`에 `sdkModel` 필드 추가, `createSdkSession`에서 저장, `discord-app.ts` lazy resume 경로 3곳에서 재전달
- 파일: `src/discord/session-store.ts`, `src/discord/channel-manager.ts`, `src/discord/interactions/select-menus.ts`, `src/discord/discord-app.ts`

### 5. SDK 업데이트

- `@anthropic-ai/claude-agent-sdk` 0.2.80 → 0.2.114

## 검증 방법

1. `/claude start-sdk` → **Opus 4.7 (1M)** 선택 → 디렉토리 선택
2. 메시지 전송 후 표시 확인: `🟢 X% ctx (X.Xk/1.0M)` — 200K가 아니라 **1.0M** 이어야 함
3. `pm2 restart sleep-discord` 후 같은 스레드에 메시지 보내기 → 여전히 1.0M 유지되는지 확인

## 주의사항

- **기존 세션은 migration 안 됨**: 패치 이전에 시작한 세션들은 `sdkModel`이 persisted mapping에 없어서 resume 시 model 없이 시작됨 → 200K로 떨어짐. 영향받은 세션은 새로 시작해야 함.
- **`@anthropic-ai/claude-agent-sdk` 0.2.114**: `supportedModels()`는 `[1m]` 변형 중 `sonnet[1m]`만 명시적으로 반환하지만, 실제로는 Opus 모델에도 `[1m]` 접미사가 유효함. CLI `/model`과 동일한 형식 사용.
- **Haiku 사이드카**: SDK가 자동으로 Haiku로 보조 연산(컴팩션 등)을 돌린다. 표시에 여러 모델이 섞여 나오는 것은 버그가 아님.
